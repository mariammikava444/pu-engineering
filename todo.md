Software Maintenance Engineering involves: **Development**, **Operation**, **Maintenance** and **Retirement**.
In other words, Software Maintenance Engineering is about achieving high quality and high productivity. We call that Q&P, aka, **Quality And Productivity**.
There are three factors that influence Q&P, which are: **People**, **Processes** and **Technology**.
## Process
___
**პროცესი** არის დინამიური ერთეული, რომელიც ასახავს შესრულებულ მოქმედებებს.

**პროცესის სპეციფიკაცია** კი არის პროცესის აღწერა, რომელიც, შეიძლება გამოვიყენოთ სხვადასხვა პროექტში იმ მიზნის მისაღწევად, რომლისთვისაც იგი იყო შემუშავებული.

**პროცესის მოდელი** განსაზღვრავს ზოგად პროცესს, რომელიც არის "ოპტიმალური" რაღაც კლასის პროექტებისთვის. ანუ, რომლის გამოყენებითად ვიღებთ მაღალ Q&P-ს.

**A process უნდა მიუთითოს რა ამოცანები და როგორ უნდა გაკეთდეს.**

**პროცესზე აქცენტირებაა რაც განასხვავებს პროგრამული უზრუნველყოფის ინჟინერიას სხვა დისციპლინებისგან.**

პროგრამული უზრუნველყოფის ინჟინერია ფოკუსირებულია 
პროდუქტების წარმოების პროცესზე.

**პროცესი არის ნაბიჯების თანმიმდევრობა რაღაც მიზნის მისაღწევად.**

პროცესი ხშირად განისაზღვრება, როგორც რაღაც ეტაპების თანმიმდევრობა.

## Software Process
___
პროცესებს რომლებიც ეხება პროგრამული უზრუნველყოფის განვითარების **ტექნიკურ (technical)** და **მართვის (management)** საკითხებს უწოდებენ პროგრამული უზრუნველყოფის პროცესს (software process).

Software Process-ში არის ორი მნიშნველოვანი კომპონენტი:
- განვითარების (development) პროცესი
- პროექტის მართვის (project management) პროცესი.

*განვითარების პროცესში* განხორციელდება ინჟინრული აქტივობები.
*პროექტის მართვის პროცესში* იგეგმება აქტივობები ისე, რომ ფასი, დრო, ხარისხი და ა.შ. გათვალისწინებული იყოს.

Software Configuration Control Process უმკავლდება პროექტის ევოლუციას და ცვლილებებს.

# Process Management Process
___
ძირითადი მიზანია პროგრამული უზრუნველყოფის გაუმჯობესება. ანუ პროცესის უნარი აწარმოოს ხარისხიანი საქონელი დაბალ ფასად გაუმჯობესებულია. ამისთვის ხდება მიმდინარე პროცესის შესწავლა.

ანუ, **პროცესის მართვის პროცესი სწავლობს მიმდინარე პროცესს, აანალიზებს მის მახასიათებლებს, არკვევს რა შიძლება გაუმჯობესდეს და დგამს შესაბამის ნაბიჯებს.**
## Software Development Process Models 
___
Development Process განსაზღვრავს დავალებებს, რომლებიც პროექტმა უნდა შეასრულოს და მათი შესრულების თანმიმდევრობას.

**პროცესი წარმართავს პროექტს და დიდ გავლენას ახდენს შედეგზე.**

პროცესის მოდელი განსაზღვრავს ზოგად პროცესს ეტაპების სახით. იგი აღწერს პროექტის ეტაპებს, მათი შესრულების თანიმდევრობას, მათი შესრულების შეზღუდვებსა და პირობებს. მისმა გამოყენებამ უნდა მოგვცეს **ღირებულების შემცირება**, **ხარისხის გაზრდა**, **ციკლის დროის შემცირება** ან სხვა სარგებელი.

პროცესის მოდელი იძლევა **ზოგად რჩევა-მითითებებს პროექტისთვის, რითაც შესაფერისი პროცესის შემუშავებას შევძლებთ.**

Development Processes:
- Waterfall Model
- Prototyping
- Iterative Development
- Timeboxing Model
### Waterfall Model
მისი ფაზები სრულდება თანმიმდევრულად, ერთმანეთის მიყოლებით.
1. Requirements
2. Design
3. Implementation
4. Verification
5. Maintenance
მინუსი ისაა, რომ ყოველი ფაზის დასრულების შემდეგ რაღაც ფისიკური შედეგი უნდა მიიღებოდეს, დოკუმენტი, რეპორტი, პროგრამა ან სხვა, ხოლო მერე მაგ პროდუქტს უნდა შემოწმება (სერტიფიკაცია, ვალიდაცია)
### Prototyping
პროტოტიპირების მოდელის დროს ჩვენ ეტაპობრივად ვქმნით პროექტის პროტოტიპულ დემო ვარიანტებს, რომელსაც მომხამერებლი ამოწმებს და შენიშვნებსაც გვეუბნება. ამიტომაც საკმარისია მცირე მოთხოვნები, არაა საჭირო სრულია სია.
### Iterative Development
უზრუნველყოფის აწყობა ხდება ინკრიმენტულად. ყოველი ინკერემენტი ამატებს რაღაც ფუნქციონალს მანამ, სანამ სრულ სისტემას არ მივიღებთ.
1. Design
2. Implementation
3. Analyze.
